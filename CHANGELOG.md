# Changelog

本文件记录仓库发布版本的用户可见变化。板端日期、计数和临时测试状态仍保存在 `docs/BOARD_TEST_LOG.md`。

## [Unreleased]

## [1.2.0] - 2026-07-17

- 将 Reference 按五类归档，示例模板按四类归档，板端短探针统一移入 `scripts/probes/`。
- 保持 `run_board_probe.py` 等公共 CLI 路径不变，由探针注册表隔离内部文件布局。
- 增加分类目录、嵌套文档链接和探针注册完整性校验，防止后续文件重新散落。
- 精简仓库 README，并统一更新 Skill、Agent 指南、测试矩阵和实测记录中的路径。

## [1.1.0] - 2026-07-17

- 增加固件能力证据表，将已测能力、置信边界、探针和回退集中管理。
- 模型包增加 nncase、目标芯片、量化方式和目标固件等转换元数据，并保持旧模型包警告级兼容。
- `run_board_probe.py` 增加安全的本地探针导出路径，raw REPL 不可用时可改由 CanMV IDE 或手动 SD 卡流程运行。
- 修复 macOS/Linux 快照打开路径，并将主机回归测试扩充到 43 项。

## [1.0.0] - 2026-07-16

- 建立分层任务路由、Reference 所有权边界和可安装包边界。
- 提供经典视觉、模型视觉、离线部署、raw REPL、UART 与 ZDT 云台工作流。
- 增加统一 RAM-only 板端探针入口、自动判读、主机回归测试和发布校验。
- 增加模板运行环境元数据、资源释放守卫、Markdown 锚点校验和三轮资源生命周期探针。
